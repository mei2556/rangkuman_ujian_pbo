Looping (Perulangan)

Apa itu Looping?

Looping atau perulangan adalah proses menjalankan suatu blok kode secara berulang-ulang sampai kondisi tertentu terpenuhi. Dengan adanya perulangan, programmer tidak perlu menuliskan kode yang sama berkali-kali sehingga program menjadi lebih singkat, rapi, dan efisien.

Dalam kehidupan sehari-hari looping dapat diibaratkan seperti perintah "tendang bola ke gawang 100 kali" atau "tendang bola sampai gol". Keduanya sama-sama melakukan pengulangan tetapi memiliki kondisi berhenti yang berbeda.

Looping Iterasi Pengulangan

Jenis-Jenis Looping

Counted Loop Uncounted Loop
Jumlah pengulangan sudah diketahui sejak awal. Jumlah pengulangan belum diketahui secara pasti.
Menggunakan FOR. Menggunakan WHILE dan DO-WHILE.
Contoh: Cetak angka 1 sampai 100. Contoh: Menunggu user memasukkan data yang benar.

FOR Loop (Counted Loop)

FOR merupakan perulangan yang digunakan ketika jumlah iterasi sudah diketahui. FOR terdiri dari tiga bagian utama yaitu inisialisasi variabel, kondisi, dan increment/decrement.

Selama kondisi bernilai true maka blok kode akan terus dijalankan. Setelah satu iterasi selesai, nilai variabel akan diperbarui sesuai increment atau decrement yang diberikan. 

for(int i = 1; i <= 5; i++){
    System.out.println(i);
}
Output :
1
2
3
4
5

WHILE Loop

WHILE digunakan ketika jumlah pengulangan belum diketahui secara pasti. Perulangan akan terus berjalan selama kondisi bernilai true.

Kondisi diperiksa terlebih dahulu sebelum blok program dijalankan. Jika kondisi false sejak awal maka isi perulangan tidak akan dieksekusi. 

int i = 1;

while(i <= 5){
    System.out.println(i);
    i++;
}
Perulangan berhenti ketika nilai i sudah lebih besar dari 5.

DO WHILE Loop

DO-WHILE hampir sama dengan WHILE, tetapi pengecekan kondisi dilakukan setelah blok kode dijalankan.

Karena kondisi dicek di akhir, maka minimal satu kali perulangan pasti akan dieksekusi walaupun kondisi bernilai false. 

int i = 1;

do{
    System.out.println(i);
    i++;
}
while(i <= 5);
DO-WHILE selalu menjalankan blok program minimal satu kali.

BREAK Statement

BREAK digunakan untuk menghentikan perulangan secara paksa sebelum kondisi utama perulangan selesai.

Biasanya digunakan ketika data yang dicari sudah ditemukan sehingga tidak perlu melanjutkan proses iterasi. 

for(int i=1; i<=10; i++){

    if(i==5){
        break;
    }

    System.out.println(i);
}
Output hanya sampai angka 4 karena perulangan dihentikan saat i bernilai 5.

CONTINUE Statement

CONTINUE digunakan untuk melewati satu iterasi tertentu dan langsung melanjutkan ke iterasi berikutnya.

Berbeda dengan BREAK yang menghentikan seluruh perulangan, CONTINUE hanya melewati iterasi yang sedang berjalan. 

for(int i=1; i<=5; i++){

    if(i==3){
        continue;
    }

    System.out.println(i);
}
Angka 3 tidak dicetak karena dilewati oleh CONTINUE.

Nested Loop

Nested Loop adalah perulangan yang berada di dalam perulangan lainnya. Setiap kali perulangan luar berjalan satu kali maka perulangan dalam akan dijalankan hingga selesai.

Nested Loop sering digunakan untuk membuat pola bintang, tabel perkalian, serta pengolahan data dua dimensi. 

for(int i=1; i<=3; i++){

    for(int j=1; j<=3; j++){
        System.out.print("* ");
    }

    System.out.println();
}
Output:
* * *
* * *
* * *

Error dan Exception Handling

Pengertian Error

Error merupakan kondisi ketika program tidak dapat berjalan sebagaimana mestinya. Java menyediakan debugger yang dapat membantu programmer mengetahui lokasi, penyebab, dan jenis error yang terjadi.

Secara umum error dibagi menjadi Syntax Error dan Runtime Error.

Syntax Error dan Runtime Error

Syntax Error

  • Kesalahan penulisan sintaks.
  • Terdeteksi saat compile.
  • File .class tidak terbentuk.
  • Contoh: lupa titik koma (;).

Runtime Error

  • Terjadi saat program dijalankan.
  • Compile berhasil.
  • File .class tetap terbentuk.
  • Contoh: pembagian dengan nol.

Exception Handling

Runtime Error dapat menyebabkan program berhenti secara tiba-tiba. Agar aplikasi tetap berjalan dengan baik, Java menyediakan mekanisme Exception Handling.

Exception Handling digunakan untuk menangani error yang terjadi saat runtime sehingga program tetap stabil dan tidak langsung berhenti.

TRY CATCH FINALLY

TRY - CATCH

TRY digunakan untuk menempatkan kode yang berpotensi menghasilkan error. Jika error terjadi maka program akan masuk ke blok CATCH. 

try{

    int hasil = 10 / 0;

}
catch(ArithmeticException e){

    System.out.println("Terjadi Error");

}
Error berhasil ditangani sehingga program tidak langsung berhenti.

Jenis Exception yang Sering Digunakan

Exception Keterangan
ArithmeticException Kesalahan operasi matematika.
NullPointerException Objek bernilai null.
NumberFormatException Kesalahan konversi String menjadi angka.
ArrayIndexOutOfBoundsException Index array melebihi batas.

TRY - CATCH - FINALLY

FINALLY adalah blok yang selalu dijalankan baik terjadi exception maupun tidak.

Biasanya digunakan untuk menutup file, koneksi database, atau membersihkan resource yang digunakan program. 

try{

    int hasil = 10 / 0;

}
catch(Exception e){

    System.out.println("Terjadi Error");

}
finally{

    System.out.println("Program Selesai");

}
Walaupun terjadi error, blok finally tetap dijalankan.
