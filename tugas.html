01

Bahasa Pemrograman

Apa itu Bahasa Pemrograman?

Kode (termasuk sintak, format, dan aturan) yang ditulis untuk membuat program aplikasi menggunakan tools tertentu. Proses penulisannya disebut Coding, dan hasilnya disebut Source Code.

Langkah Membuat Aplikasi

DesignUML, Flowchart, ERD, Mockup
CodingBahasa Pemrograman + Code Editor
CompilingCompiler / Interpreter
AplikasiKasir, Games, Mobile App, dll.

Klasifikasi Berdasarkan Generasi Teknologi

Gen. 1 — Bahasa Mesin

Biner (0 dan 1) murni. Tidak lagi digunakan untuk membuat program saat ini.

Gen. 2 — Assembler

Bahasa tingkat rendah. Masih dipakai untuk firmware, BIOS, dan driver hardware.

Gen. 3 — Tingkat Tinggi

Mendekati bahasa manusia. Contoh: Pascal, COBOL, C. Diterjemahkan via compiler/interpreter.

Gen. 4 — Pengembangan dari Gen. 3, dengan tambahan interaksi database. Contoh: Java, Python, PHP, C++, VB, SQL.

Klasifikasi Berdasarkan Code Base

Bahasa Induk Ciri Khas Turunan Utama
Bahasa C Kurung kurawal { }, titik koma ;, fungsi main() C++, Java, C#, Golang, JavaScript, PHP
Bahasa Pascal Begin … End;, deklaratif di awal program Lazarus, Delphi, Embarcadero, Python
Bahasa Basic Blok kode dengan sintak pembuka/penutup, pindah baris sebagai pemisah Visual Basic, VB.NET

Klasifikasi Berdasarkan Platform

Desktop

Kompleks, akses hardware langsung. Contoh: Kasir, Office Suite, ERP.

Bahasa: C/C++, Java, Delphi, VB

Mobile

Untuk Smartphone & Tablet (Android/iOS). Contoh: Medsos, Games.

Bahasa: Java/Kotlin, Swift, Flutter, React Native

Web

Diakses via browser, lintas platform. Contoh: Website, Google Suite.

Front End: HTML, CSS, JS · Back End: PHP, Python, Node.JS

02

Paradigma Pemrograman

Definisi Paradigma

Pendekatan atau gaya dalam menulis dan menyusun kode program berdasarkan prinsip tertentu. Paradigma mempengaruhi cara berpikir programmer dalam merancang program dan membantu memilih bahasa yang tepat. Tidak ada paradigma yang benar/salah — yang ada adalah metode mana yang lebih efektif untuk kasus tertentu.

Analogi Indomie: Tujuan yang sama (membuat makanan dari mie), tetapi cara yang ditempuh berbeda — itulah "paradigma".

Imperative Programming

Berfokus pada BAGAIMANA komputer melakukan tugas melalui instruksi langkah demi langkah. Programmer harus menjelaskan setiap langkah karena ada tujuan akhir yang jelas.

Procedural Object Oriented Parallel Processing

Contoh app: Kasir, Akunting, Games bertingkat, Kalkulator
Bahasa: Java, C, C++, JavaScript

Declarative Programming

Menekankan pada APA yang harus dilakukan, bukan bagaimana. Programmer mendeklarasikan aturan/data; program menjawab sesuai aturan tersebut saat ditanya.

Logic Functional Database

Contoh app: ChatGPT, SQL Query
Bahasa: SQL, Prolog, Haskell

Procedural vs Object Oriented Programming (OOP)

Aspek Procedural OOP
Pendekatan Berbasis prosedur/fungsi, tujuan akhir jelas Berbasis objek yang meniru dunia nyata
Struktur Data Sekumpulan prosedur yang mengoperasikan data Kelas yang membentuk objek (atribut + method)
Data & Fungsi Data bersifat global, dilewatkan antar fungsi Data terenkapsulasi dalam objek
Keamanan Data Kurang terlindungi (dapat diakses dari mana saja) Lebih terlindungi (akses dikontrol via method)
Alur Program Linear, berurutan dari awal ke akhir Objek-objek saling berinteraksi
Konsep Utama Fungsi, Prosedur Encapsulation, Inheritance, Polymorphism
Contoh Bahasa C, Pascal, Python (mode prosedural) Java, C++, Python (mode OOP)

Pengenalan Java

Sejarah Singkat Java

1991 — Proyek GreenPatrick Naughton & James Gosling merancang perangkat dengan memori minimal yang bisa berjalan di berbagai hardware.
Pengaruh BahasaSintaks dari C++ · Teknologi Virtual Machine dari Pascal (Niclaus Wirth)
Nama Awal: OAKKemudian berganti nama menjadi Java.

Mengapa Java?

Simple

OOP

Distributed
Banyak library terintegrasi Java Networking
Multi-Platform
Architecture Neutral berkat JVM
Robust
Compiler teliti + Exception Handling
Dynamic
Penambahan method tidak ganggu program berjalan
GC
Garbage Collection: optimasi memori tidak terpakai
Security
Menyediakan 3 lapis code security

Tiga Komponen Utama Java

JDK — Java Development Kit

Paket lengkap untuk pengembang. Berisi JRE + Compiler (javac) + Debugger + Tools lainnya. Diperlukan untuk menulis & mengkompilasi kode Java.

JRE — Java Runtime Environment

Lingkungan runtime berisi JVM + Library & API standar. Cukup untuk menjalankan aplikasi Java tanpa mengembangkan.

JVM — Java Virtual Machine

Mesin virtual yang menjalankan bytecode Java. Melakukan JIT Compilation (bytecode → kode mesin). Bersifat platform-independent.

Hierarki: JDK ⊃ JRE ⊃ JVM. JVM ada di dalam JRE, sehingga jika hanya ingin menjalankan program cukup install JRE.

Fase Pemrograman Java

Tulis kode di Text Editor / IDESimpan sebagai NamaKelas.java — nama file harus sama dengan nama class.

Jalankan dengan javajava NamaKelas → JVM mengeksekusi bytecode, program berjalan.

Jenis Error dalam Java

⚠ Syntax Error

  • Kesalahan penulisan sintak
  • Terdeteksi saat proses compile
  • File .class tidak akan terbentuk
  • Contoh: lupa tanda ;, salah kapitalisasi

🔴 Runtime Error

  • Kesalahan logika program
  • Terdeteksi saat fungsi dijalankan
  • Compiling berhasil, file .class tetap terbentuk
  • Contoh: pembagian dengan 0 (10/0)

Instalasi Java JDK

Langkah-langkah Instalasi JDK di Windows

Unduh Installer JDK Kunjungi oracle.com/java/technologies/downloads. Pilih versi LTS (disarankan Java 17 atau Java 21). Pilih OS (Windows) → pilih x64 Installer.
Proses Instalasi Buka file .exe → klik Next → perhatikan lokasi instalasi (biasanya C:\Program Files\Java\jdk-xx) → klik Next hingga selesai → Close.
Verifikasi Instalasi Buka Command Prompt → ketik java --version dan javac -version. Jika muncul versi Java → instalasi berhasil!
Setting PATH (jika langkah 3 gagal) Start → My Computer → Properties → Advanced System Settings → Environment Variables → Edit variabel Path → tambahkan dua entri baru: 
C:\Program Files\Java\jdk-21
C:\Program Files\Java\jdk-21\bin
Setelah setting PATH, beberapa PC perlu restart. Selanjutnya ulangi langkah verifikasi (Step 3).

Checklist Siap Mulai Java

✔ JDK terinstall di C:\Program Files\Java\ ✔ PATH sudah diatur (java + javac) ✔ java --version menampilkan versi
✔ Pilih editor (Notepad++ atau IDE) ✔ Buat file NamaKelas.java ✔ Compile & jalankan Hello World!
