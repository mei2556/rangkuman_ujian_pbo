01
ENCAPSULATION
Penyembunyian data (data hiding) dengan membungkus variabel dan metode ke dalam satu unit — yaitu Class.

Tujuan

  • Melindungi data dari perubahan langsung luar kelas
  • Mengontrol bagaimana data diakses/dimodifikasi
  • Meningkatkan keamanan & kestabilan program

Metode Enkapsulasi

  • Buat Class, semua atribut & method di dalamnya
  • Lakukan kontrol Access Modifier
  • Sediakan getter & setter yang bersifat public

Access Modifier

Mengatur tingkat akses terhadap kelas, atribut, metode, atau konstruktor.

Modifier Dalam Kelas Paket Sama Subclass Di Luar Paket
private ✓ ✗ ✗ ✗
(default) ✓ ✓ ✗ ✗
protected ✓ ✓ ✓ ✗
public ✓ ✓ ✓ ✓
02
INHERITANCE
Mekanisme OOP yang memungkinkan subclass mewarisi properti dan metode dari superclass.
MANUSIA — Superclass
↓
Dosen
Mahasiswa

Tujuan

  • Reusability — kode tidak ditulis ulang
  • Organisasi kode menjadi hierarki
  • Mendukung polimorfisme

Cara Penulisan (Java)

  • Buat superclass secara normal
  • Subclass menggunakan kata kunci extends
  • Subclass mewarisi semua atribut & method non-private
  • Tambahkan atribut/method spesifik di subclass

Method Overriding

Mendefinisikan ulang method superclass di subclass dengan perilaku baru yang berbeda.

Nama method sama Parameter sama Return type sama @Override
03
POLYMORPHISM
Konsep OOP yang memungkinkan objek memiliki banyak bentuk tergantung konteks penggunaannya.
poly = banyak · morph = bentuk

Tujuan

Reusability — kode lebih fleksibel Scalability — mudah ditambah Interoperability — method bisa dipanggil tanpa tahu kelas aslinya
METHOD OVERRIDING

Mendefinisikan ulang method yang diwariskan dari superclass.

  • Terjadi antar superclass & subclass
  • Nama, parameter, return type harus sama
  • Gunakan @Override annotation
METHOD OVERLOADING

Beberapa method dengan nama sama, parameter berbeda, dalam satu kelas.

  • Nama method sama
  • Jumlah / tipe / urutan parameter berbeda
  • Digunakan saat input bisa bermacam tipe
01
LIBRARY
Kumpulan kelas, metode, dan fungsi yang disediakan untuk mempermudah pengembangan aplikasi. Library berisi kode yang dapat digunakan ulang tanpa perlu menulis dari nol.

Mengapa Penting?

  • Efisiensi — hemat waktu dan usaha
  • Modularitas — pakai fungsionalitas tanpa tahu cara implementasinya
  • Reusabilitas — bisa dipakai di berbagai proyek

Struktur Library Java

  • Library terdiri dari banyak Package
  • Package berisi banyak Class
  • Class berisi banyak Method
  • Contoh: java.util → Scanner → nextInt()

Standard Library

  • Library bawaan Java
  • java.lang, java.util, java.io
  • java.lang otomatis diload — tidak perlu import

External Library

  • Library pihak ketiga / buatan sendiri
  • Contoh: Apache Commons, Gson, JUnit
  • Harus ditambahkan secara manual ke proyek

Membuat Library Sendiri

Pisahkan class library dari class utama (main). Simpan di file/folder berbeda, lalu panggil dari program utama.

Pisah file library Class terpisah dari Main Panggil via import / package
02
PACKAGE
Cara untuk mengelompokkan kelas yang terkait — seperti folder dalam sistem file. Digunakan untuk mengorganisir kode dan mencegah konflik nama kelas.

Manfaat Package

  • Pengelompokan class secara logis (utils, model, controller)
  • Menghindari konflik nama class
  • Kontrol akses (public, default)

Jenis Package

  • Built-in — bawaan Java (java.util, java.io, java.lang)
  • User-defined — package yang kita buat sendiri

Langkah Membuat User-defined Package

  1. Buat folder baru di tempat program utama disimpan (nama folder = nama package)
  2. Letakkan file library ke dalam folder tersebut
  3. Tambahkan package nama_package; di baris pertama file library
  4. Compile class/package: javac nama_package/NamaClass.java
  5. Buat file utama (main program), panggil dengan import nama_package.NamaClass;
  6. Compile program utama dan jalankan

Built-in Package Populer

java.util → Scanner, Random java.io → BufferedReader, Console java.lang → Math, String, System (auto-load)

Daftar lengkap: docs.oracle.com/en/java/javase/11/docs/api

01
GUI — GRAPHICAL USER INTERFACE
Antarmuka pengguna berbasis grafis — tombol, teks, jendela, dan menu — yang memudahkan pengguna berinteraksi dengan program.

Library GUI di Java

  • AWT — lebih kuno, berbasis sistem operasi
  • Swing — lebih modern & fleksibel (javax.swing) ← yang dipelajari
  • JavaFX — paling baru, mendukung animasi & desain kaya

Komponen Utama Swing

  • JFrame — jendela utama program
  • JLabel — teks statis / output
  • JTextField — input teks satu baris
  • JButton — tombol yang bisa diklik
  • FlowLayout — menyusun komponen kiri-ke-kanan
02
MEMBUAT PROGRAM GUI (SWING)
Program GUI dibuat dengan membuat Frame, menambahkan komponen, lalu menangani interaksi lewat Event Handler.

Langkah-langkah

  1. Buat Frame — JFrame frame = new JFrame("Judul"), atur ukuran dengan setSize(w,h)
  2. Set Close Operation — frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE) agar program benar-benar berhenti saat jendela ditutup
  3. Set Layout — frame.setLayout(new FlowLayout()) untuk menyusun komponen otomatis
  4. Tambah Komponen — buat objek JLabel, JTextField, JButton, lalu frame.add(...)
  5. Tampilkan Frame — frame.setVisible(true)
  6. Tambah Event Handler — agar tombol bisa merespons klik pengguna

Import yang Dibutuhkan

javax.swing.JFrame javax.swing.JLabel javax.swing.JTextField javax.swing.JButton java.awt.FlowLayout java.awt.event.ActionListener java.awt.event.ActionEvent
03
EVENT HANDLER
Digunakan untuk menjalankan kode ketika terjadi suatu peristiwa (event), misalnya saat pengguna menekan tombol.
Klik Tombol Tekan Keyboard Gerakan Mouse Timer Selesai Fokus Berpindah

Tiga Konsep Utama

  • Event — peristiwa yang terjadi (mis. tombol diklik)
  • Listener — objek yang mendengarkan event
  • Handler / Callback — kode yang dijalankan saat event terjadi

Cara Penulisan

  • button.addActionListener(...) — daftarkan listener ke tombol
  • new ActionListener() { ... } — buat objek listener anonim
  • actionPerformed(ActionEvent e) — method yang dijalankan saat diklik
  • Di dalam actionPerformed: ambil data dari TextField, tampilkan ke Label
